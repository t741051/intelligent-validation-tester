## 二、架構哲學：與前端對齊的三層分離

Django 的原生架構是 **View → Model**，但對於有複雜業務邏輯的系統（驗證流程、連接器、分數計算），我們採用**增強版三層架構**，跟前端哲學一致：

```
HTTP Request
    │
    ▼
┌──────────────────────┐
│  View Layer (DRF)    │  薄控制器
│  ViewSet / APIView   │  只做：權限、序列化、呼叫 Service
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  Service Layer       │  業務邏輯核心
│  services.py         │  交易、外部系統協調、業務規則
└──────────┬───────────┘
           │
           ├─► Model (Django ORM)           # 資料存取
           ├─► Connector (O-RAN / 外部)     # 外部系統
           └─► Celery Task                  # 非同步任務
```

### 分層規則

**View Layer（ViewSet）**：
- ✅ 處理 HTTP 請求/回應
- ✅ 權限檢查、序列化
- ✅ 呼叫 Service 並回傳結果
- ❌ 禁止直接寫業務邏輯
- ❌ 禁止跨多個 Model 的交易

**Service Layer（`services.py`）**：
- ✅ 所有業務邏輯
- ✅ 資料庫交易（`@transaction.atomic`）
- ✅ 協調多個 Model、Connector、Task
- ❌ 禁止處理 HTTP（不知道 request/response）

**Model Layer（Django ORM）**：
- ✅ 資料結構、關聯、索引
- ✅ 簡單 queryset 方法
- ❌ 不做業務規則

### Django App 劃分原則

- **每個 App 對應一個業務領域**，內部自足（models + views + services + serializers）
- **跨 App 的邏輯放 `core/` 或 `validations/`（作為協調者）**
- **`connectors/` 獨立不依賴 Django 框架**，方便測試

## 四、Mock / 正式版切換（連接器）

跟前端 Service 雙軌對稱，後端 Connector 也走 Factory 模式：

```python
# apps/connectors/factory.py
from django.conf import settings
from .o1_netconf import O1NetconfConnector
from .o1_mock import O1MockConnector
from .a1_rest import A1RestConnector
from .a1_mock import A1MockConnector

def get_o1_connector(dut):
    if settings.USE_MOCK_CONNECTORS:
        return O1MockConnector(dut)
    return O1NetconfConnector(dut)

def get_a1_connector(dut):
    if settings.USE_MOCK_CONNECTORS:
        return A1MockConnector(dut)
    return A1RestConnector(dut)
```

`.env`:
```
USE_MOCK_CONNECTORS=false   # 預設 — 接真機(O1 ncclient / A1 httpx)
USE_MOCK_CONNECTORS=true    # 開發 / 沒有實機時
```

```python
class Site(models.Model):
    class Region(models.TextChoices):
        DOMESTIC = 'domestic', '國內'
        INTERNATIONAL = 'international', '國外'

    class Environment(models.TextChoices):
        # 場域的室內/室外屬性。DUT 不再自己存 environment,
        # 而是透過 FK 綁到某個 Site,環境由場域決定。
        # Serializer 會把 site.name / site.environment 帶到 DutSerializer 當 read-only 欄位。
        INDOOR = 'indoor', '室內'
        OUTDOOR = 'outdoor', '室外'

    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
    name = models.CharField(max_length=200)
    region = models.CharField(max_length=32, choices=Region.choices)
    environment = models.CharField(
        max_length=16, choices=Environment.choices, default=Environment.INDOOR,
    )
    address = models.CharField(max_length=500, blank=True)
    location = models.JSONField(null=True, blank=True)        # {lat, lng} 保留備用,新 UI 不使用
    floor_plan_url = models.CharField(max_length=500, blank=True)  # 實體地圖底圖 URL,空值則用網格底
    created_at = models.DateTimeField(auto_now_add=True)


class BaseStation(models.Model):
    """場域內的網元(gNB / SMO / RIC 三種)。對應 UI「網元列表」與「邏輯拓樸」「實體地圖」。

    - `position` 為實體地圖上的相對座標 {x, y} 0-100%,可拖拉定位(僅 gNB 會顯示在實體地圖)
    - `geo` 保留欄位備用但目前 UI 不使用(改採相對座標)
    - 邏輯拓樸採自動分層(SMO 上層 → RIC 中層 → gNB 下層),不讀 position
    - `config` JSONField 存類型專屬設定:
      - gNB: gnb_id / plmn_id / tac / cell_count
      - SMO / RIC: endpoint_url / interfaces / api_token
    """

    class NodeType(models.TextChoices):
        GNB = 'gnb', 'gNB'
        SMO = 'smo', 'SMO'
        RIC = 'ric', 'RIC'

    class Status(models.TextChoices):
        NORMAL = 'normal', '正常'
        WARNING = 'warning', '需要關注'
        OFFLINE = 'offline', '離線'

    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
    site = models.ForeignKey(Site, on_delete=models.CASCADE, related_name='stations')
    code = models.CharField(max_length=64)          # 場域內唯一識別碼,例 'gnb-001'
    name = models.CharField(max_length=200)         # 顯示名稱,例 '北側 gNB'
    node_type = models.CharField(max_length=16, choices=NodeType.choices)
    mgmt_ip = models.GenericIPAddressField(null=True, blank=True)
    mgmt_port = models.IntegerField(null=True, blank=True)
    vendor = models.CharField(max_length=100, blank=True)
    model = models.CharField(max_length=100, blank=True)
    config = models.JSONField(default=dict, blank=True)   # 類型專屬欄位
    position = models.JSONField(default=dict)              # {x, y} 相對座標 0-100%
    geo = models.JSONField(null=True, blank=True)          # {lat, lng}(保留備用)
    status = models.CharField(max_length=16, choices=Status.choices, default=Status.NORMAL)
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        indexes = [models.Index(fields=['site', 'node_type'])]
        unique_together = [('site', 'code')]


class TopologyLink(models.Model):
    """網元間的拓樸連線(供邏輯拓樸圖繪製)。"""
    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)

class Camera(models.Model):

# apps/duts/models.py
class Dut(models.Model):
    """統一的 DUT 模型,涵蓋 SMO / RIC / xApp / rApp 四種類型。

    type = models.CharField(max_length=16, choices=Type.choices)
    endpoint = models.CharField(max_length=500)           # 允許 IP / IP:port / URL(建立時不做連線測試)
    interfaces = models.JSONField(default=list)           # ['O1', 'A1', 'E2']
    status = models.CharField(max_length=16, choices=Status.choices, default='offline')
    response_time_ms = models.IntegerField(null=True, blank=True)
    data_format = models.CharField(max_length=32, blank=True)
    last_check = models.DateTimeField(null=True, blank=True)
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        indexes = [models.Index(fields=['type', 'status'])]


class DataQualityBaseline(models.Model):
    """資料品質驗證所需的基準設定 + 門檻值,1:1 掛在 SMO/RIC 類型的 DUT 上。

    xApp / rApp 不做資料品質驗證(它們走智慧程度驗證),透過 `limit_choices_to`
    在 Admin / Serializer 層限制只允許掛到 SMO 或 RIC。

    註冊流程(FE 流程):使用者在「資料品質驗證」頁按「添加 DUT」 → 選一個未註冊的
    SMO/RIC DUT → 為它建立一筆 baseline(= 註冊)。之後列表只顯示有 baseline 的 DUT。

    欄位變動:v1 的 `required_fields` 已移除(不再用欄位清單判完整性),改用下列
    三個關聯欄位(分類 / AI cases / 測試情境)。門檻值(timeliness/completeness/accuracy)
    保留給驗證引擎計分用,但 **註冊時不必指定**,會套用 model 預設值,之後再編輯。
    """
    class TestCategory(models.TextChoices):
        UNDERGROUND = 'underground', '地下樓層'
        GROUND_FLOOR = 'ground-floor', '地面樓層'
        HIGH_FLOOR = 'high-floor', '高樓層'

    dut = models.OneToOneField(
        Dut,
        on_delete=models.CASCADE,
        related_name='data_quality_baseline',
        limit_choices_to={'type__in': ['SMO', 'RIC']},
    )
    test_category = models.CharField(                 # 場域類別(地下/地面/高樓)
        max_length=32, choices=TestCategory.choices, blank=True,
    )
    supported_ai_cases = models.JSONField(default=list)   # 此 DUT 支援的 AI case 名稱列表,
                                                           # 例 ['CCO', 'ES', 'Load Balance', 'QoE']
    test_scenarios = models.ManyToManyField(           # 關聯至具體測試情境
        'scenarios.TestScenario',
        blank=True,
        related_name='data_quality_baselines',
    )
    timeliness_max_lag_sec = models.IntegerField(default=60)  # 資料延遲上限(秒)
    min_completeness = models.FloatField(default=90.0)    # 0-100 完整性門檻
    min_accuracy = models.FloatField(default=85.0)        # 0-100 準確性門檻
    notes = models.TextField(blank=True)
    updated_at = models.DateTimeField(auto_now=True)
```

```python
# apps/duts/views.py
from rest_framework import viewsets, status
from rest_framework.decorators import action
from rest_framework.response import Response
from .models import Dut
from .serializers import DutSerializer, DutInputSerializer
from .services import DutService
from core.permissions import HasPermission

class DutViewSet(viewsets.ModelViewSet):
    queryset = Dut.objects.select_related('site').all()
    serializer_class = DutSerializer
    filterset_fields = ['type', 'status', 'site']
    search_fields = ['name', 'endpoint']
    ordering_fields = ['created_at', 'last_check']

    def get_permissions(self):
        if self.action in ['create', 'update', 'partial_update', 'destroy']:
            return [HasPermission('dut:write')]
        if self.action in ['test_interface', 'healthcheck_all']:
            return [HasPermission('dut:write')]
        return [HasPermission('dut:read')]

    @action(detail=True, methods=['post'], url_path='test-interface')
    def test_interface(self, request, pk=None):
        """單一 DUT 即時檢查(也作為列表列的 🔄 刷新按鈕來源)"""
        dut = self.get_object()
        interfaces = request.data.get('interfaces', [])
        result = DutService.trigger_interface_test(dut, interfaces, user=request.user)
        return Response(result, status=status.HTTP_200_OK)

    @action(detail=False, methods=['post'], url_path='healthcheck-all')
    def healthcheck_all(self, request):
        """整頁「刷新狀態」按鈕觸發 — 對目前所有 DUT 跑一輪 health check"""
        for dut in Dut.objects.all():
            try:
                DutService.trigger_interface_test(dut, dut.interfaces or [], user=request.user)
            except Exception:
                continue
        return Response({'ok': True}, status=status.HTTP_200_OK)
```

## 七、Service Layer 範例

```python
# apps/validations/services.py
from django.db import transaction
from celery import current_app
from .models import ValidationRun
from .tasks import run_validation

class ValidationService:

    @classmethod
    @transaction.atomic
    def start_run(cls, scenario_id, target_type, target_id, user):
        scenario = TestScenario.objects.select_for_update().get(id=scenario_id)
        target = cls._get_target(target_type, target_id)
        cls._authorize(user, target)

        run = ValidationRun.objects.create(
            scenario=scenario,
            target_type=target_type,
            target_id=target.id,
            triggered_by=user,
            status='pending',
        )
        task = run_validation.delay(str(run.id))
        run.celery_task_id = task.id
        run.save(update_fields=['celery_task_id'])
        return run

    @classmethod
    def cancel_run(cls, run_id, user):
        run = ValidationRun.objects.get(id=run_id)
        cls._authorize(user, run)
        if run.status not in ('pending', 'running'):
            raise ValueError('Run is not cancellable')
        if run.celery_task_id:
            current_app.control.revoke(run.celery_task_id, terminate=True)
        run.status = 'cancelled'
        run.ended_at = timezone.now()
        run.save(update_fields=['status', 'ended_at'])
        return run
```

```python
class OverviewService:

    @classmethod
    def summary(cls):

            'kpis': cls._kpis(),
            'distribution': cls._distribution(),
            'platform_status': cls._platform_status(),
            'recent_runs': cls._recent_runs(),

            ),

        }
```

```python
from channels.generic.websocket import AsyncJsonWebsocketConsumer

class ValidationRunConsumer(AsyncJsonWebsocketConsumer):
    async def connect(self):
        self.run_id = self.scope['url_route']['kwargs']['run_id']
        self.group_name = f'run_{self.run_id}'
        user = self.scope['user']
        if user.is_anonymous or not await self._can_view(user, self.run_id):

            return
        await self.channel_layer.group_add(self.group_name, self.channel_name)
        await self.accept()

    async def disconnect(self, code):
        await self.channel_layer.group_discard(self.group_name, self.channel_name)

    # Channel layer event handlers
    async def run_progress(self, event):
        await self.send_json({'type': 'progress', 'payload': event['payload']})

    async def run_log(self, event):
        await self.send_json({'type': 'log', 'payload': event['payload']})

    async def run_metric(self, event):
        await self.send_json({'type': 'metric', 'payload': event['payload']})

    async def run_completed(self, event):
        await self.send_json({'type': 'completed', 'payload': event['payload']})
        await self.close()
```

```python
class ValidationStrategy(ABC):
    @abstractmethod
    def run(self, run, publish) -> dict:

# apps/validations/strategies/data_validation.py
#
# 適用範圍：僅 SMO / RIC DUT（xApp/rApp 走 IntelligenceValidationStrategy）
# 分數與門檻來自 dut.data_quality_baseline（1:1 OneToOne）
class DataValidationStrategy(ValidationStrategy):
    def __init__(self, dut, connector):
        assert dut.type in ('SMO', 'RIC'), 'DataValidation 僅支援 SMO / RIC'
        self.dut = dut
        self.baseline = dut.data_quality_baseline   # 無 baseline 則在呼叫端先擋下
        self.connector = connector

    def run(self, run, publish):
        steps = [
            ('connectivity', self._check_connectivity, 25),
            ('completeness', self._check_completeness, 50),
            ('accuracy', self._check_accuracy, 75),
            ('timeliness', self._check_timeliness, 100),

        metrics = {}
        for key, fn, pct in steps:
            value = fn()
            metrics[key] = value
            publish(progress=pct, metric=(key, value))
        overall = weighted_overall(metrics, self.baseline)
        return {'score': overall, 'metrics': metrics}
```

```python
from celery import shared_task
from .models import ValidationRun
from .strategies import get_strategy
from apps.reports.services import ReportService

@shared_task(bind=True)
def run_validation(self, run_id):
    run = ValidationRun.objects.get(id=run_id)
    run.status = 'running'
    run.save(update_fields=['status'])
    publish_progress(run_id, 0, 'running')

    try:
        strategy = get_strategy(run)
        result = strategy.run(
            run,
            publish=lambda progress, metric=None: (
                publish_progress(run_id, progress, 'running'),
                publish_metric(run_id, *metric) if metric else None,
                record_metric(run, *metric) if metric else None,
            ),
        )
        run.score = result['score']
        run.status = 'passed' if result['score'] >= 80 else 'failed'
    except Exception as e:
        run.status = 'failed'
        logger.exception(f'Run {run_id} failed: {e}')
    finally:
        run.ended_at = timezone.now()
        run.save()

    report_url = ReportService.generate(run)
    run.report_url = report_url
    run.save(update_fields=['report_url'])
    publish_completed(run_id, run.score, report_url)
```

### 13.2 RBAC

```python
# core/permissions.py
from rest_framework.permissions import BasePermission

ROLE_PERMISSIONS = {
    'admin': {'*'},
    'platform_vendor': {'dut:*', 'baseline:*', 'validation:run', 'report:read'},
    'app_vendor': {'application:*', 'validation:run', 'report:read'},
    'viewer': {'*:read'},
}

def has_permission_for(role, permission):
    perms = ROLE_PERMISSIONS.get(role, set())
    if '*' in perms:
        return True
    resource, action = permission.split(':')
    return permission in perms or f'{resource}:*' in perms or f'*:{action}' in perms

class HasPermission(BasePermission):
    def __init__(self, required):
        self.required = required
    def has_permission(self, request, view):
        return has_permission_for(request.user.role, self.required)
```

- HTTPS only（production）
- Django Admin：限 admin 角色 + IP 白名單
- Secret 管理：env 變數 + Vault（production）
